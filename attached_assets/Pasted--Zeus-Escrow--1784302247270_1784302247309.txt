Задача: Интеграция Zeus Escrow и управляющего дашборда в экосистему

Нам нужно создать законченный продукт, который будет включать в себя:

1. Брендинг и навигация
Сайт (/): Главная страница должна быть лендингом, кратко описывающим, что такое Zeus Insurance и Zeus Escrow. Это общая "витрина" для всех продуктов.

Навигация: В хедере сайта должны быть разделы: Insurance (страхование) и Escrow (эскроу). Каждый раздел ведёт на свой дашборд.

Дашборд Escrow (/escrow): Это отдельная страница с интерфейсом для управления эскроу-сделками. Её дизайн должен соответствовать общему стилю сайта (тёмная тема, золотые акценты).

2. Дашборд Escrow
На этой странице должны быть реализованы следующие функции и роли:

Для Инициатора (Участник):
Создание сделки: Форма с полями: Executor (адрес), Amount (сумма в USDC), Timeout (время в секундах), Proof (опционально, хэш). Кнопка "Deposit and Create".

Просмотр своих сделок: Таблица со всеми созданными им сделками. Статусы: Active, Fulfilled, Refunded, Expired. Кнопка "Request Refund" для активных сделок по истечении таймаута.

Для Исполнителя (Организатор):
Просмотр сделок для исполнения: Таблица со сделками, где он указан как Executor. Статусы: Active, Fulfilled, Refunded, Expired.

Подтверждение исполнения: Кнопка "Confirm Execution" для активных сделок, которая открывает диалог для ввода Proof. После подтверждения сделка переходит в статус Fulfilled, и средства перечисляются исполнителю.

Для Администратора (Вы):
Управление ролями: Возможность присваивать пользователям роли (Admin, Partner, Investor, Oracle) через отдельную панель.

Генерация API-ключей: Возможность генерировать уникальные API-ключи для внешних систем (оракулов, партнёров).

Полный обзор: Просмотр всех сделок и общей статистики.

3. Техническая реализация
Контракт: Использовать существующий ZeusEscrowBOT.sol с хранением proof.

База данных: Использовать существующий PostgreSQL (через Drizzle) для хранения ролей, API-ключей и логов всех действий.

API: Добавить эндпоинты для создания, подтверждения, получения списка сделок и управления ролями.
